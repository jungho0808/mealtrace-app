# MealTrace 기획 문서

## 컨셉

MealTrace는 현재 위치 주변에서 직접 가서 먹을 식당 하나를 추천하고, 식사 후 만족/보통/불만족 평가를 다음 추천에 반영하는 미니멀 음식 선택 앱입니다.

## 문제 정의

식사 장소를 고를 때 사용자는 너무 많은 선택지, 리뷰 비교, 지도 탐색 때문에 결정 피로를 겪습니다. MealTrace는 모든 후보를 보여 주기보다, 지금 상황에 맞는 하나의 선택지를 제안해 빠르게 결정할 수 있게 돕습니다.

## 핵심 방향

- 추천은 하나만 제공한다.
- 직접 방문 가능한 식당만 다룬다.
- 추천 결과는 필요한 정보만 보여 준다.
- 식사 후 간단한 평가를 받아 다음 추천에 반영한다.
- 지도 탐색은 외부 지도 앱에 맡긴다.

## v1 목표

- 사용자가 앱을 열고 짧은 시간 안에 식당 하나를 결정할 수 있다.
- 사용자가 선택한 식당 기록을 남길 수 있다.
- 사용자가 다음 실행 시 이전 식사 만족도를 평가할 수 있다.
- 평가 결과가 이후 추천에 영향을 줄 수 있는 기본 구조를 갖춘다.

## v1 범위

포함 항목:

- 현재 위치 기반 주변 식당 추천
- 직접 방문 식당만 추천
- 추천 결과 1개 표시
- 식당명, 음식 종류, 예상 가는 시간, 예상 가격 표시
- `이걸로 할게요` 선택 시 기록 저장
- 외부 지도 앱으로 이동
- 미평가 식사 평가 요청
- 만족/보통/불만족/안 먹었어요 평가 저장

제외 항목:

- 배달
- 포장
- 앱 내 지도
- 리뷰 비교
- 복수 후보 비교
- 상세 리뷰 작성
- 소셜 기능

## 추천 원칙

v1 추천은 복잡한 탐색보다 결정 보조에 집중합니다.

- 사용자의 현재 위치에서 갈 수 있는 식당을 우선한다.
- 최근 만족도가 낮았던 음식 종류나 식당은 추천 우선순위를 낮출 수 있다.
- `안 먹었어요`는 만족도 평가로 보지 않고 선택 취소 또는 미방문 기록으로 다룬다.
- 사용자가 반복적으로 만족한 음식 종류나 가격대는 다음 추천에 긍정적으로 반영할 수 있다.

## 기록할 데이터 초안

- 선택한 식당명
- 음식 종류
- 예상 가는 시간
- 예상 가격
- 선택 시각
- 평가 상태
- 평가 값: 만족, 보통, 불만족, 안 먹었어요

## 성공 기준

- 사용자가 추천 화면에서 추가 탐색 없이 식당을 선택할 수 있다.
- 앱 재실행 시 미평가 식사를 자연스럽게 평가할 수 있다.
- 평가가 다음 추천에 반영된다는 제품 경험이 명확하다.

## 향후 확장 후보

- 선호 음식 종류 설정
- 예산 범위 설정
- 혼밥/동행 여부 반영
- 날씨나 시간대 반영
- 추천 제외 식당 관리
- 방문 기록 통계
