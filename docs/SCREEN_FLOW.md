# MealTrace 화면 흐름

## v1 기본 흐름

```text
앱 실행
  |
  |-- 미평가 식사가 있음
  |     |
  |     v
  |   식사 평가 화면
  |     |
  |     v
  |   추천 화면
  |
  |-- 미평가 식사가 없음
        |
        v
      추천 화면
        |
        v
      추천 결과 확인
        |
        v
      "이걸로 할게요" 선택
        |
        v
      선택 기록 저장
        |
        v
      외부 지도 앱 이동
```

## 1. 앱 실행

앱을 실행하면 먼저 미평가 식사가 있는지 확인합니다.

미평가 식사가 있으면 추천 화면보다 평가 화면을 먼저 보여 줍니다. 미평가 식사가 없으면 바로 추천 화면으로 이동합니다.

## 2. 식사 평가 화면

조건:

- 이전에 `이걸로 할게요`를 눌러 선택한 식사가 있음
- 아직 만족도 평가를 하지 않음

화면 내용:

- 이전에 선택한 식당명
- 음식 종류
- 평가 선택지

평가 선택지:

- 만족
- 보통
- 불만족
- 안 먹었어요

동작:

- 만족/보통/불만족 선택 시 평가를 저장하고 다음 추천에 반영
- 안 먹었어요 선택 시 미방문으로 기록하고 만족도 신호로는 반영하지 않음
- 평가 완료 후 추천 화면으로 이동

## 3. 추천 화면

현재 위치 주변에서 직접 방문 가능한 식당 하나를 추천합니다.

화면 내용:

- 식당명
- 음식 종류
- 예상 가는 시간
- 예상 가격
- `이걸로 할게요` 버튼

v1에서는 추천 후보 목록, 리뷰 비교, 앱 내 지도는 표시하지 않습니다.

## 4. 선택 완료 및 외부 지도 앱 이동

사용자가 `이걸로 할게요`를 누르면 선택한 식당을 기록합니다.

저장 내용:

- 식당명
- 음식 종류
- 예상 가는 시간
- 예상 가격
- 선택 시각
- 평가 상태: 미평가

기록 저장 후 외부 지도 앱으로 이동합니다.

## 예외 흐름

### 위치 권한이 없는 경우

현재 위치 기반 추천을 할 수 없으므로 위치 권한 요청 화면 또는 안내 상태를 보여 줍니다.

### 주변 추천 후보가 없는 경우

추천 가능한 식당이 없다는 상태를 보여 주고, 사용자가 다시 시도할 수 있게 합니다.

### 외부 지도 앱 이동 실패

선택 기록은 유지하고, 지도 앱을 열 수 없다는 안내를 보여 줍니다.
